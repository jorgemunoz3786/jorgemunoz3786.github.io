Somos dos alumnos del colegio Sant Josep Obrer, de 1º del CFGS de Desarrollo de Aplicaciones Multiplataforma. Este es

un proyecto para poder comprender el funcionamiento de GitHub Pages. Este proyecto en particular consiste en una página

web online que accede a PokeAPI (Una base de datos que almacena un conjunto de perfiles de Pokémon) y, mediante una

barra de búsqueda, se muestra en tu navegador la entrada del Pokémon en particular. Además, hemos añadido una

estructura similar a la de la Pokédex; .



Ya que se trata de un añadido del propósito de este repositorio, PokedexWeb se trata de una página de entretenimiento

en la que poder localizar y identificar las características de los Pokémon.

### **1 - USO DE POKEDEXWEB**

Para utilizar la web de PokedexWeb, todo lo que es necesario es entrar en el link "[https://jorgemunoz3786.github.io](https://jorgemunoz3786.github.io)",

que mostrará en tu pantalla una interfaz similar a la Pokédex. En el centro de la pantalla de la Pokédex, hay una barra

de búsqueda en la que poder introducir información. El placeholder de esta barra es "Introduce el nombre o número de

Pokémon... \[1-1025]", esto se debe a que, en esta barra, se puede introducir tanto el nombre del Pokémon como su número

identificativo, lo que equivale a todos los Pokémon hasta la novena generación. la información que contiene es:

* Nombre
* Foto representativa
* Tipos \[Entre 1 y 2]
* Estadísticas \[HP, Ataque, Defensa, Ataque especial, Defensa especial y Velocidad]
* Altura \[en metros] y peso \[kg]
* Habilidades
* Descripción breve



Además de la barra de búsqueda, el logo de Pokédex de la parte superior y el botón inferior redondo recargan la página,

eliminando el perfil abierto.

### **2 - CONTRIBUCIÓN**

Este proyecto nació como un proyecto de prueba para poder comprender el funcionamiento de GitHub Pages y practicar con

la PokeAPI, así que cualquier aportación que ayude a mejorar la claridad del código, la interfaz o la organización del

proyecto es bienvenida.



Si quieres contribuir:

* Haz un fork del repositorio.
* Crea una rama nueva con tu mejora: \[git checkout -b mejora-nombre]
* Asegúrate de que el código sigue siendo fácil de leer y mantiene el enfoque didáctico.
* Haz un pull request explicando qué has cambiado y por qué.



Ideas de contribución:

* Mejorar la UI de la Pokédex.
* Añadir filtros o buscador de Pokémon.
* Añadir más datos de la PokeAPI (evoluciones, versiones MEGA…).
* Añadir un método de búsqueda (Por tipo, Por generación...).
* Optimizar las llamadas a la API o el render.
* Corregir errores o mejorar la accesibilidad.



No hay normas estrictas: el objetivo es aprender juntos y mantener el proyecto ligero y comprensible.
